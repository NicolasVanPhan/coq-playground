Q: What if you introduce n > 0 early, next to n ?

We must use return annotation to help Rocq understand that n > 0 becomes 0 > 0 in the first case.
However, annotation cannot bind :
  • the discriminee
  • with the type of a variable already in scope (which is the case if pf is declared early)
So we must introduce pf after the match-cases

Definition mypred_strong1' (n : nat) : n > 0 → nat :=
  match n return n > 0 → nat with
  | 0 ⇒ (fun pf : 0 > 0 ⇒ (match zgtz pf with end))
  | S n' ⇒ (fun _ ⇒ (n'))
  end.

Q: Re-implement pred_v2 with subset types ?


Print sig.
Locate "{ _ : _ | _ }".

Definition mypred_strong2 (s : {n : nat | n > 0}) : nat :=
  match s with
  | exist _ O pf ⇒ match zgtz pf with end
  | exist _ (S n') _ ⇒ n'
  end.

Q: Why is the extracted Ocaml code with subset types ?

Subset type is :
  • argument
  • proof about it
The proof being erased, only 1 thing remains, the actual argument. Now, without the proof, sig has :
  • 1 constructor exist
  • with 1 argument n (and not two since P is out)
Since inductive types of 1 ctor with 1 argument are useless, they don't bring more information than the argument itself, there's an optimization pass to erase them.
That's why the resulting ocaml code is exactly the same as before.

6.2 - Decidable Proposition Types


Q: Define eq_nat_deciable : {n = m} + {n ≠ m}


Definition eq_nat_decidable_raw : ∀ n m : nat, {n = m} + {n ≠ m}.
  refine(
    fix f (n m : nat) :=
    match n, m with
    | O, O ⇒ left (eq_refl O)
    | S n', S m' ⇒ (match f n' m' with
      | left _ ⇒ left _
      | right _ ⇒ right _
      end
    )
    | _, _ ⇒ right _
    end);

    try (intro Hcontra; discriminate || congruence).

    try (match goal with
    | [ H : n' = m' |- _] ⇒ rewrite H
    end; reflexivity).

Defined.

Eval compute in eq_nat_decidable_raw 2 2.
Eval compute in eq_nat_decidable_raw 2 3.

6.3 - Partial Subset Types


Q: Define The maybe type


Inductive maybe (A : Set) (P : A → Prop) : Set :=
| Unknown : maybe P
Alternative : | Found (x : A) (H : P x) : maybe P
| Found : ∀ x : A, P x → maybe P
.

Notation "{{ x | P }}" := (maybe (fun x ⇒ P)).
Notation "??" := (Unknown _).
Notation "[| x |]" := (Found _ x _).

6.5 - A Type-Checking Example

The idea here is to illustrate above techniques on a certified type-checker for a language with arithmetic and boolean expressions.

Inductive exp : Set :=
| E_Nat : nat → exp
| E_Plus : exp → exp → exp
| E_Bool : bool → exp
| E_And : exp → exp → exp
.

Inductive typ : Set :=
| T_Nat
| T_Bool
.

Inductive hasType : exp → typ → Prop :=
| HT_Nat : ∀ n : nat,
    hasType (E_Nat n) T_Nat
| HT_Plus : ∀ e1 e2 : exp,
    hasType e1 T_Nat
 → hasType e2 T_Nat
 → hasType (E_Plus e1 e2) T_Nat
| HT_Bool : ∀ b : bool,
    hasType (E_Bool b) T_Bool
| HT_And : ∀ e1 e2 : exp,
    hasType e1 T_Bool
 → hasType e2 T_Bool
 → hasType (E_And e1 e2) T_Bool
.

Definition eq_type_dec : ∀ t1 t2 : typ, {t1 = t2} + {t1 ≠ t2}.
  decide equality.
Defined.

Notation "e1 ;; e2" := (if e1 then e2 else ??)
(right associativity, at level 60).

Locate "{{ _ | _ }}". Print maybe.
Locate "[| _ |]". Locate "??".
Print hasType.

Definition typeCheck : ∀ e : exp, {{ t | hasType e t }}.
  Hint Constructors hasType.
  refine( fix F (e : exp) : {{ t | hasType e t }} :=
    match e return {{t | hasType e t}} with
    | E_Nat _ ⇒ [| T_Nat |]
    | E_Plus e1 e2 ⇒
      t1 <-- F e1;
      t2 <-- F e2;
      eq_type_dec t1 T_Nat;;
      eq_type_dec t2 T_Nat;;

      
      [| T_Nat |]
    | E_Bool _ ⇒ [| T_Bool |]
    | E_And e1 e2 ⇒
      t1 <-- F e1;
      t2 <-- F e2;
      eq_type_dec t1 T_Bool;;
      eq_type_dec t2 T_Bool;;
      [| _ |]
    end
  );
  try subst;
  try constructor;
  try assumption.
Defined.

Lemma hasType_det : ∀ e t1, hasType e t1 →
  ∀ t2, hasType e t2 → t1 = t2.
Proof.
    induction 1; inversion 1; reflexivity.
Qed.

Definition typeCheck' : ∀ e : exp, {t | hasType e t} + {∀ t, ¬ hasType e t}.
  Hint Constructors hasType.
  Hint Resolve hasType_det.
  refine(
    fix F (e : exp) :=
      match e return {t | hasType e t} + {∀ t, ¬ hasType e t} with
      | E_Nat _ ⇒ [|| T_Nat ||]
      | E_Plus e1 e2 ⇒
        t1 <--- F e1;
        t2 <--- F e2;
        eq_type_dec t1 T_Nat;;;
        eq_type_dec t2 T_Nat;;;
        [|| T_Nat ||]
      | E_Bool _ ⇒ [|| T_Bool ||]
      | E_And e1 e2 ⇒
        t1 <--- F e1;
        t2 <--- F e2;
        eq_type_dec t1 T_Bool;;;
        eq_type_dec t2 T_Bool;;;
        [|| T_Bool ||]
      end
  )
  ; clear F; crush' tt hasType; eauto.
Defined.
